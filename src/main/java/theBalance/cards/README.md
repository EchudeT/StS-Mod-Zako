## ⚠️ 重要注意事项

### 1. 战斗津贴系统
**战斗津贴不是直接的金币**，而是通过 `CombatGoldPower` 追踪的独立数值：
- 战斗中获得，战斗结束后消失
- 使用战斗津贴的卡牌：回扣、保险合同、库存清理、恶魔期权、高息存储等
- 流动资产卡牌会消耗战斗津贴来保持格挡

### 2. 复杂机制实现

#### 时空套利 (TimeArbitrage)
- **实现方式：** 玩家连续行动2个正常回合，第3回合能量为0无法行动
- **技术细节：** 使用 `TimeArbitragePower` 追踪回合状态，惩罚回合通过设置能量为0实现

#### 逆流 (Countercurrent)
- **实现方式：** 检测并保存所有负面效果（虚弱、易伤、脆弱、负力量、负敏捷）
- **转换逻辑：** 立即移除负面并转为正面，回合结束时给敌人双倍原始负面效果
- **技术细节：** 使用内部 `PowerData` 类存储转换前的状态

#### 净值突刺 (NetWorthThrust)
- **实现方式：** 通过 `NetWorthTrackingPower` 追踪本回合获得正面buff的次数
- **攻击逻辑：** 基础攻击1次 + 每获得1次buff额外攻击1次
- **重置机制：** 每回合结束自动重置计数

### 3. 简化实现的效果

以下效果使用了简化实现，可能需要根据实际游戏测试进行调整：

- **镜像投射** - 直接读取敌人意图伤害值
- **绝对中立** - 简单的生命值调整，未处理护盾等复杂情况
- **资产置换** - 简单的易伤层数交换
- **无限膨胀** - 使用变量追踪伤害倍率
- **InsuranceContract（保险合同）** - 简化为直接给战斗津贴，未实现"格挡被打破"的判定
